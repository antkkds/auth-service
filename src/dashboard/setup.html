🚀 Setup Guide

Step-by-step instructions for beginners. Copy these samples directly.

Prerequisites

What You Need Before Starting

  • ☐ A Railway account (free at railway.com) — or any Node.js host
  • ☐ A PostgreSQL database (Railway gives you one free)
  • ☐ Node.js 20+ installed on your machine (for local testing)
  • ☐ A GitHub account to clone the repo
Step 1

Clone the Repository

Terminal
git clone https://github.com/antkkds/auth-service.git
cd auth-service
npm install

💡 Make sure you have Node.js 20+ installed first.

Step 2

Set Up Environment Variables

Copy the sample env file and fill in your values:

.env — Copy this into your project
# Server
PORT=3005
BETTER_AUTH_URL=http://localhost:3005
FRONTEND_URL=http://localhost:5173

# Security — GENERATE YOUR OWN with: openssl rand -base64 32
BETTER_AUTH_SECRET=replace-with-your-32-char-secret-key-here!!

# Database — get this from Railway after creating a PostgreSQL DB
DATABASE_URL=postgresql://postgres:password@your-railway-host:5432/auth_db
🔑 Generate a secret key: 
openssl rand -base64 32

Paste the output as your BETTER_AUTH_SECRET.

🐘 Get your PostgreSQL URL from Railway:
  1. Go to Railway dashboard → New Project → Provision PostgreSQL
  2. Click on the PostgreSQL service → Connect tab
  3. Copy the DATABASE_URL connection string
  4. Paste it into your .env file
Step 3

Run Database Migrations

Better Auth auto-creates the required tables (users, sessions, etc.):

Terminal
npx auth@latest generate   # Generate SQL schema
npx auth@latest migrate    # Apply to your database

💡 This creates tables: user, session, account, verification

Step 4

Start the Auth Server

Development mode
npm run dev

You should see:

🔐 Auth service running at http://localhost:3005
   Auth API at http://localhost:3005/api/auth/ok

Verify it's working:

curl http://localhost:3005/api/auth/ok
# → { "ok": true }
Step 5

Connect Your Frontend (Sample Code)

Add this to your React / Vue / Svelte frontend:

Install the client SDK
npm install better-auth
lib/auth-client.ts — Create this file
import { createAuthClient } from "better-auth/client"

export const authClient = createAuthClient({
  baseURL: "http://localhost:3005" // Your auth server URL
})
Example: Sign Up Form
import { authClient } from "./lib/auth-client"

async function handleSignUp(email, password, name) {
  const { data, error } = await authClient.signUp.email({
    email,
    password,
    name,
  })

  if (error) {
    console.error("Sign up failed:", error.message)
    return
  }

  console.log("User created:", data.user)
  // Session cookie is set automatically
}
Example: Check if User is Logged In
import { authClient } from "./lib/auth-client"

const { data: session } = await authClient.getSession()
if (session) {
  console.log("Logged in as:", session.user.name)
} else {
  console.log("Not logged in")
}
Step 6

Deploy to Railway

One-click deployment:

  1. Push your repo to GitHub (already done ✅)
  2. Go to railway.app → New Project → Deploy from GitHub repo
  3. Select antkkds/auth-service
  4. Add a PostgreSQL database (Railway provisions it)
  5. Set environment variables in Railway dashboard:
    • BETTER_AUTH_SECRET ← your secret key
    • BETTER_AUTH_URL ← https://your-app.railway.app
    • FRONTEND_URL ← https://your-frontend.com
  6. Railway auto-detects the build command and starts the server

💡 Railway automatically sets DATABASE_URL for you when you add PostgreSQL.

⚠️

Troubleshooting

Failed to initialize database adapterCheck your DATABASE_URL in .env
BETTER_AUTH_SECRET is not setGenerate one with openssl rand -base64 32
404 on API endpointsMake sure you're hitting /api/auth/* paths
CORS errors from frontendSet FRONTEND_URL to match your frontend's origin
